# litert-lm OpenAI Proxy — Raspberry Pi 5

Runs **Gemma 4 E2B IT** locally on a Pi 5 via Google's
[LiteRT](https://ai.google.dev/edge/litert) runtime and exposes an
OpenAI-compatible `/v1/chat/completions` endpoint.

- Model loads once at startup and stays in memory (no subprocess per request)
- Real token streaming via `send_message_async` — not fake chunking
- systemd user service with auto-restart

**Hardware:** Raspberry Pi 5 (aarch64, 8 GB RAM)  
**Model:** [`litert-community/gemma-4-E2B-it-litert-lm`](https://huggingface.co/litert-community/gemma-4-E2B-it-litert-lm) — 2.4 GB  
**Runtime:** `litert-lm` v0.10.1 + `litert-lm-api` Python package

## Quick install (recommended)

Clone the repo, then run `install-proxy-and-cloudflare-tunnel.sh`. It handles everything in one shot.

```bash
git clone <repo-url> ~/Downloads/src/llm/litert_openai_proxy
cd ~/Downloads/src/llm/litert_openai_proxy
./install.sh
```

or:

```bash
./install-proxy-and-cloudflare-tunnel.sh
```

`install-proxy-and-cloudflare-tunnel.sh` will:
1. Install `litert-lm` CLI via `uv tool install`
2. Download and import the Gemma 4 E2B IT model (~2.4 GB)
3. Create a Python venv with `fastapi`, `uvicorn`, `pydantic`, `litert-lm-api`
4. Write a systemd user service pointing at this directory
5. Create/update a Cloudflare tunnel and DNS route
6. Write and start a user systemd service for the tunnel

After install, the proxy is running at `http://0.0.0.0:8000/v1` and will
start automatically on login/reboot.

If this is your first tunnel on a machine, run `cloudflared tunnel login`
once before running the installer.

### Installer options

You can override defaults with environment variables:

```bash
CF_TUNNEL_NAME=pecan-litert-openai \
CF_HOSTNAME=pecan.ggg.ad \
OPENAI_API_KEY=your-secret \
./install-proxy-and-cloudflare-tunnel.sh
```

Useful flags:
- `SKIP_DNS=true` — skip `cloudflared tunnel route dns`
- `DRY_RUN=true` — print actions without applying changes

## Manual setup

If you prefer to run each step yourself:

### 1. Install prerequisites

```bash
# uv (if not already installed)
curl -LsSf https://astral.sh/uv/install.sh | sh

# litert-lm CLI
uv tool install litert-lm
```

### 2. Download the model

```bash
litert-lm import \
  --from-huggingface-repo litert-community/gemma-4-E2B-it-litert-lm \
  gemma-4-E2B-it.litertlm \
  gemma-4-e2b-it
```

### 3. Run the proxy with uv

The run script manages its own venv using uv — no manual activation needed.

```bash
cd ~/Downloads/src/llm/litert_openai_proxy
./run-litert-openai-proxy.sh
```

On first run it creates `~/.venvs/litert-openai-proxy/` and installs
`fastapi`, `uvicorn`, `pydantic`, and `litert-lm-api` into it, then starts
uvicorn. Subsequent runs skip the install step if packages are up to date.

To override the venv location:

```bash
VENV_PATH=/path/to/venv ./run-litert-openai-proxy.sh
```

#### One-liner with `uv run` (no persistent venv)

```bash
cd ~/Downloads/src/llm/litert_openai_proxy
uv run --with fastapi --with uvicorn --with pydantic --with litert-lm-api \
  uvicorn litert_openai_proxy:app --host 0.0.0.0 --port 8000
```

This creates a temporary environment each run. Slower to start, but requires
no venv management.

### 4. Set up the systemd service manually

```bash
# Copy the service template and edit paths if needed
cp litert-openai-proxy.service ~/.config/systemd/user/

# Reload and enable
systemctl --user daemon-reload
systemctl --user enable --now litert-openai-proxy
```

## Testing

> **Note:** `curl` segfaults on Pi 5 when the litert native `.so` files are
> loaded in-process (ASLR conflict). Use `python3` or `wget` instead.

```bash
# Health check
python3 -c "
import urllib.request
print(urllib.request.urlopen('http://localhost:8000/healthz').read().decode())
"

# Non-streaming
python3 -c "
import urllib.request, json
req = urllib.request.Request(
  'http://localhost:8000/v1/chat/completions',
  data=json.dumps({
    'model': 'gemma-4-e2b-it',
    'messages': [{'role': 'user', 'content': 'What is the capital of France?'}]
  }).encode(),
  headers={'Content-Type': 'application/json'},
  method='POST'
)
resp = json.loads(urllib.request.urlopen(req, timeout=60).read())
print(resp['choices'][0]['message']['content'])
"

# Streaming
python3 -c "
import urllib.request, json
req = urllib.request.Request(
  'http://localhost:8000/v1/chat/completions',
  data=json.dumps({
    'model': 'gemma-4-e2b-it',
    'stream': True,
    'messages': [{'role': 'user', 'content': 'Count to 5'}]
  }).encode(),
  headers={'Content-Type': 'application/json'},
  method='POST'
)
with urllib.request.urlopen(req, timeout=60) as r:
  for line in r:
    line = line.decode().strip()
    if line.startswith('data: ') and line != 'data: [DONE]':
      chunk = json.loads(line[6:])
      print(chunk['choices'][0]['delta'].get('content', ''), end='', flush=True)
print()
"
```

## Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `MODEL_ID` | `gemma-4-e2b-it` | Model ID from `litert-lm list` |
| `HOST` | `0.0.0.0` | Bind address |
| `PORT` | `8000` | Port |
| `LITERT_BACKEND` | `cpu` | `cpu` or `gpu` (VideoCore VII on Pi 5) |
| `OPENAI_API_KEY` | _(unset)_ | If set, Bearer token auth is required |
| `VENV_PATH` | `~/.venvs/litert-openai-proxy` | Python venv location |

Set these in the service with `systemctl --user edit litert-openai-proxy`:

```ini
[Service]
Environment=LITERT_BACKEND=gpu
Environment=OPENAI_API_KEY=your-secret
```

## Architecture

### Model lifecycle

The model loads once when the FastAPI app starts (`lifespan` context) and
stays in memory for the life of the process.  Each request creates a fresh
`Conversation` from the persistent `Engine` — no subprocess, no reload.

```
uvicorn process
└── litert_lm.Engine  (loaded at startup, ~500 MB RSS)
    └── Conversation  (created per request, destroyed after)
        └── send_message_async → MessageIterator → SSE chunks
```

### Streaming

`send_message_async` returns a synchronous `MessageIterator` that yields
tokens as they are generated by the C++ runtime.  The proxy bridges this to
FastAPI's `StreamingResponse` via a background thread and an asyncio-safe
queue, keeping the event loop unblocked during inference.

### Multi-turn history

The OpenAI API is stateless — clients send the full message history on every
request.  Prior turns are replayed through the `Conversation` object (each
prior user turn runs inference to advance KV-cache state).  This is a no-op
for single-turn requests (the common case) and adds latency proportional to
history length for multi-turn sessions.

### GPU backend (experimental)

```bash
export LITERT_BACKEND=gpu
./run-litert-openai-proxy.sh
```

Pi 5 VideoCore VII GPU support in litert-lm is experimental — fall back to
`cpu` if you see errors.
